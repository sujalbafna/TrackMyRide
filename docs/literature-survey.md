# Literature Survey: Smart Transportation and Vehicle Tracking Systems

This table provides a survey of existing literature and systems related to real-time vehicle tracking, IoT integration in transportation, and route management. It serves to contextualize "Track My Ride" within the current research landscape and highlight its unique contributions.

| Ref. No. | Title / Author(s)                                                              | Methodology / Technology Used                                                              | Key Contribution / Finding                                                                                                 | Limitations & Gaps Addressed by "Track My Ride"                                                                                                   |
| :------- | :----------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------ |
| **[1]**  | "Real-Time Vehicle Tracking System using GPS and Firebase" (Gupta et al.)        | Utilizes GPS modules on vehicles to send coordinate data to a Firebase Realtime Database. An Android app visualizes the location. | Demonstrates a foundational model for live tracking using accessible IoT components and a serverless backend.                      | Lacks role-based dashboards, route management, and advanced features like payment processing or AI optimization. Focus is purely on tracking.     |
| **[2]**  | "A Real-time School Bus Tracking System using Node.js and Firebase" (Le et al.)    | Implements a system with a GPS device, a central Node.js server, and Firebase for data synchronization and notifications. | Provides a more robust architecture with a dedicated server, enabling more complex logic and push notifications for arrivals.       | The architecture is not fully serverless, requiring server maintenance. Lacks integrated payments, attendance, and AI-driven features. |
| **[4]**  | "An IoT Based Smart System for Real-time School Bus Monitoring" (Chavan et al.)   | Focuses on hardware integration, using RFID for student check-in/check-out and sending SMS alerts to parents.                 | Innovates in automated attendance tracking and provides a direct, albeit basic, communication channel (SMS) to parents.           | Relies on specific hardware (RFID) which can be costly to scale. Lacks a unified web platform, route optimization, and digital payments.       |
| **[7]**  | "Applying Heuristic Algorithms for School Bus Routing Problem" (Ho et al.)      | A theoretical approach using heuristic algorithms (e.g., Tabu Search) to solve the complex school bus routing problem (SBRP). | Provides an effective computational model for optimizing routes based on static constraints like capacity and time windows.       | The solution is theoretical and not integrated into a real-time system. It does not use modern AI and cannot adapt to dynamic traffic data. |
| **Our Work** | **"Track My Ride"**                                                              | **Next.js, Firebase (Auth, Firestore), Google AI (Genkit), Google Maps API, ShadCN UI.**      | **A holistic, cloud-native platform integrating live tracking, role-based access, automated notifications, digital payments, and AI-powered route optimization into a single, scalable, and user-friendly interface.** | **Addresses the limitations of previous systems by providing a fully integrated, serverless solution with advanced features and a modern UX.**    |
